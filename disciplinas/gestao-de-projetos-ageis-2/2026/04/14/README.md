# Gestão de Projetos - 2026-04-14

## Tecnicas de Elicitação

Técnicas utilizadas para identificar, coletar e documentar os requisitos do projeto junto aos stakeholders:

- Entrevistas: Conversas diretas com clientes, usuários e especialistas para entender necessidades e expectativas

- Workshops: Sessões colaborativas com múltiplos participantes para definir requisitos em conjunto

- Brainstorming: Geração criativa de ideias e soluções para problemas identificados

- Questionários: Formulários estruturados para coletar informações de um grande número de pessoas

- Observação: Análise do contexto e ambiente de trabalho para identificar processos e necessidades reais

- Prototipos: Criação de mockups ou versões preliminares para validar entendimento dos requisitos

## Documento de Requisitos de Software (DRS)

Documento formal que consolida todos os requisitos levantados na fase de elicitação:

- Requisitos Funcionais: Descrevem o que o sistema deve fazer, funcionalidades específicas e casos de uso

- Requisitos Não-Funcionais: Definem qualidades como performance, segurança, usabilidade e escalabilidade

- Critérios de Aceitação: Estabelecem as condições necessárias para que um requisito seja considerado completo

- Rastreabilidade: Mapeia origem e evolução de cada requisito ao longo do projeto

- Priorização: Classifica requisitos por importância e urgência para o desenvolvimento
